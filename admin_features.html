OGC Compliant SDI

Essential components for deploying OGC SDI

GeoNode includes the essential building blocks than an agency needs to deploy an OGC compliant SDI. Having GeoServer as a foundation enables you to publish data from a wide variety of spatial data sources using Open Standards from the OGC and ISO. GeoServer is the reference implementation of WFS and WCS and is a certified compliant WMS.

GeoNode's Metadata and Cataloguing system also enables you to publish metadata in a variety of geospatial metadata standards (ISO, FGDC, NASA DIF, Dublin Core). pycsw is certified OGC compliant and a CSW reference implementation, providing a standards-based metadata and catalogue component of SDI.
Features include:

  • Web Map Service (WMS)
  • Web Feature Service (WFS)
  • Web Coverage Service (WCS)
  • Catalogue Service for Web (CSW)
  • Web Map Context (WMC)
  • Tile Map Service (TMS)

Metadata

Metadata: Catalogue, Authoring and Management

Metadata is a key component of any SDI, and a key component of GeoNode's data and map publishing workflow. GeoNode provides a complete metadata catalogue solution. Users and adminstrators can use GeoNode's metadata editor or upload an existing geospatial metadata XML document. Even further, default metadata is scraped from each individuals profile. Records are then editable through the web and accessible via search or a variety of standards-based catalogue interface standards via pycsw (OGC CSW, SRU, OpenSearch). Users can also access dataset and map metadata in a variety of standard formats (ISO, FGDC, NASA DIF, Dublin Core).
Features Include:

  • CSW
  • Upload
  • Search
  • Format Support
  • Versioning

Security

Security Framework

GeoNode provides a robust framework for ensuring that it is is both easy to share your data with other users and groups as well as ensuring that data that is meant to be secure and private is only accessible to authorized users. GeoNode builds on Django's authentication framework and integrates with GeoServer's security subsystem. Permissions can be applied to layers and maps and can be used to assign read, write and edit permissions to them. Additionally, GeoNode's integration with GeoServers security system ensures that the Web Services provied from your GeoNode honor the security setup you and your users have defined. The GeoNode development team is also working to make it easy to use external services for authentication such as oAuth and LDAP as well as against social platforms like Facebook and Twto make.

  • Generic Permissions
  • Layer and Map object Permissions
  • Integration with GeoServer security at the service level
  • Integration with external authentication services

Scalability

Scalability, reliability

Since GeoNode's web components are based on django, the path to scalabilty is well travelled as Django is used on some of the highest traffic sites in the world. Caching can be implemented at several levels including at the page and layer level. Additionally GeoNode leverages GeoWebCache to either dynamically or statically cache web map tiles locally or to a CDN which dramatically improves performance for your users and makes it easy to run high traffic geonode sites with minimal demands on your infrastructure. The GeoNode development team is also constantly working to make it easier and easier to deploy GeoNode to cloud based services such as Amazon Web Services (AWS) or OpenShift and making scalability on those platforms automatic and frictionless.

Administration Console

Django Administrator Console

GeoNode provides a customized version of the default Django Management or Adminsitration console which provides an easy to use interface for looking at the entire set of modules and data that make up a GeoNode and to perform bulk operations against those. The Admin console can also be used to do basic usage monitoring and to monitor user provided content for acceptability and xxx per your sites specific use case.

  • Metadata Editing
  • Bulk Operations
